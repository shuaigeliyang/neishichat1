## 🔴 高优先级问题（已修复）

### 问题1：意图识别错误导致调用错误接口

**位置：** `frontend/src/utils/intentDetector.js`

**问题描述：**
- 用户问："那我的学生挂科了会怎么办"
- 系统识别为：数据库查询（个人数据）
- 正确识别应为：文档问答（政策规定）

**根本原因：**
意图识别器看到"我的"就判断为数据库查询，但忽略了"挂科"和"会怎么样"这些政策咨询关键词。

**修复方案：**
1. 添加明确的数据库查询模式（我的+具体数据项）
2. 添加政策咨询关键词优先级
3. 调整优先级顺序

**修复状态：** ✅ 已修复并测试通过

**验证方法：**

### 建议1：添加性能监控

**位置：** 全局

**建议：**
```javascript
// 添加性能监控
const performanceMonitor = {
  startTime: Date.now(),
  endpoints: {
    '/api/rag/answer': 0,
    '/api/chat': 0,
    '/api/intelligent/query': 0
  },

  record(endpoint, duration) {
    if (!this.endpoints[endpoint]) {
      this.endpoints[endpoint] = [];
    }
    this.endpoints[endpoint].push(duration);
  },

  getStats(endpoint) {
    const times = this.endpoints[endpoint] || [];
    if (times.length === 0) return null;

    const avg = times.reduce((a, b) => a + b, 0) / times.length;
    const max = Math.max(...times);
    const min = Math.min(...times);

    return { avg, max, min, count: times.length };
  }
};
```

```javascript
const detectedIntent = intentStrategies
```

### 分析2：错误处理一致性

**发现：**
- `handleDatabaseQuery` 有错误处理
- `handleDocumentQuery` 有错误处理
- `handleFormQuery` 有错误处理
- `handleFormGenerate` 有错误处理
- `handleChatQuery` 也有错误处理

**评价：** ✅ 错误处理覆盖较好

**改进建议：**
统一错误处理模式，减少重复代码：

```javascript
const handleApiCall = async (apiCall, fallbackCall, context) => {
  try {
    const response = await apiCall();
    return response;
  } catch (error) {
    console.error(`❌ ${context}失败:`, error);

    // 通知用户
    message.warning(`${context}服务暂时不可用，正在使用备用方案...`);

    // 执行降级方案
    if (fallbackCall) {
      return await fallbackCall();
    }
  }
};

// 使用
await handleApiCall(
  () => axios.post('/api/rag/answer', { question }),
  () => handleChatQuery(question),
  '文档查询'
);
```
