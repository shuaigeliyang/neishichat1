## 🎯 问题分析

### 当前状态

**存在的核心问题：**

1. **无对话记忆**
   - 每次对话都是独立的
   - AI无法记住之前说过的话
   - 不支持连续的多轮对话

2. **无法追问**
   - 用户无法基于之前的回答进行追问
   - 例如："那具体怎么申请？" → AI不知道在问什么

3. **sessionId利用不足**
   - 虽然有sessionId标记会话
   - 但只在数据库保存时使用
   - 没有用于维护对话上下文

4. **历史记录存而不用**
   - chat_history表保存了所有对话
   - 但在调用AI时没有读取和使用

### 影响范围

| 功能模块 | 当前问题 | 影响 |
|---------|---------|------|
| 普通聊天 | 无记忆 | 用户体验差，无法连续对话 |
| RAG问答 | 无上下文 | 无法追问细节 |
| 智能查询 | 无记忆 | 需要重复描述需求 |

### 用户场景示例

#### ❌ 当前体验（无上下文）

```
用户："转专业需要什么条件？"
AI："根据第三章第六条规定，需要符合以下条件..."
```

## 🎨 技术方案设计

### 方案概述

采用**混合上下文管理策略**，针对不同场景使用不同的上下文策略：

```
┌─────────────────────────────────────────────┐
│          上下文管理器 (Context Manager)       │
├─────────────────────────────────────────────┤
│                                             │
│  ┌─────────────┐  ┌─────────────┐          │
│  │ 短期上下文   │  │ 长期上下文   │          │
│  │ Short-term  │  │ Long-term   │          │
│  ├─────────────┤  ├─────────────┤          │
│  │ 最近N轮对话  │  │ 话题摘要     │          │
│  │ 3-5轮      │  │ 关键实体     │          │
│  │ 用于连续对话 │  │ 用户偏好     │          │
│  └─────────────┘  └─────────────┘          │
│                                             │
│  ┌─────────────┐  ┌─────────────┐          │
│  │ RAG上下文   │  │ 意图上下文   │          │
│  │ RAG        │  │ Intent      │          │
│  ├─────────────┤  ├─────────────┤          │
│  │ 当前话题     │  │ 当前意图     │          │
│  │ 相关文档块   │  │ 查询类型     │          │
│  │ 便于引用     │  │ 上下文感知   │          │
│  └─────────────┘  └─────────────┘          │
│                                             │
└─────────────────────────────────────────────┘
```

### 三层上下文架构

#### 1. **短期上下文（Short-term Context）**

**用途：** 保存最近N轮完整对话，用于连续对话

**数据结构：**
```javascript
{
  sessionId: "1234567890",
  userId: 1,
  messages: [
    { role: 'user', content: '...', timestamp: '...' },
    { role: 'assistant', content: '...', timestamp: '...' },
    // ... 最近3-5轮对话
  ],
  maxTurns: 5,
  expiry: 1800000  // 30分钟过期
}
```

**适用场景：**
- 连续问答
- 追问细节
- 多步骤操作

#### 2. **长期上下文（Long-term Context）**

**用途：** 保存话题摘要和关键信息，用于长期记忆

**数据结构：**
```javascript
{
  userId: 1,
  sessionId: "1234567890",
  summary: {
    currentTopic: "转专业",          // 当前讨论的话题
    mentionedEntities: [             // 提到的实体
      { type: 'policy', name: '转专业管理办法' },
      { type: 'page', num: 162 }
    ],
    userPreferences: {               // 用户偏好
      preferredStyle: '简洁',
      topics: ['转专业', '奖学金']
    }
  },
  lastUpdated: '2026-03-25T11:00:00Z'
}
```

**适用场景：**
- 跨会话记忆
- 个性化推荐
- 上下文关联

#### 3. **RAG上下文（RAG Context）**

**用途：** 保存当前RAG问答的相关文档块，便于引用

**数据结构：**
```javascript
{
  sessionId: "1234567890",
  currentQuery: "转专业需要什么条件？",
  relevantChunks: [
    {
      chunkId: 1234,
      page: 162,
      chapter: "第三章 申请条件",
      text: "...",
      score: 0.85
    }
    // ... Top-K相关文档块
  ],
  queryHistory: [
    // ... 历史查询记录
  ]
}
```

**适用场景：**
- 文档问答
- 追问细节
- 引用来源

### 上下文策略

| 场景类型 | 上下文策略 | 历史轮数 | 过期时间 |
|---------|-----------|---------|---------|
| 普通聊天 | 短期上下文 | 5轮 | 30分钟 |
| RAG问答 | RAG上下文 + 短期 | 3轮 | 1小时 |
| 数据查询 | 意图上下文 | 2轮 | 15分钟 |
| 多轮操作 | 短期 + 长期 | 10轮 | 2小时 |

## 🔐 权限控制增强

### 当前权限问题

**存在的安全隐患：**

1. **数据访问无限制**
   - 学生可能通过特殊查询获取他人信息
   - 下载全学院数据没有权限验证
   - 缺少细粒度的权限控制

2. **查询权限模糊**
   - 教师可能访问不该看的学生数据
   - 管理员权限过于宽泛
   - 缺少操作审计日志

3. **表单下载无分类**
   - 所有表单对所有人可见
   - 没有基于角色的表单访问控制
   - 缺少下载次数统计

### 权限矩阵设计

```
┌──────────────────────────────────────────────────────────┐
│                    权限控制矩阵                            │
├──────────────────────────────────────────────────────────┤
│  功能模块         │  学生  │  教师  │  管理员  │           │
├──────────────────────────────────────────────────────────┤
│  查看个人信息      │   ✅   │   ✅   │    ✅    │  仅自己   │
│  查看成绩信息      │   ✅   │   ⚠️  │    ✅    │  学生仅自己│
│  查看班级学生      │   ❌   │   ✅   │    ✅    │  仅本班级 │
│  查看学院数据      │   ❌   │   ⚠️  │    ✅    │  需授权   │
│  下载个人数据      │   ✅   │   ✅   │    ✅    │  仅自己   │
│  下载班级数据      │   ❌   │   ✅   │    ✅    │  仅本班级 │
│  下载学院数据      │   ❌   │   ❌   │    ✅    │  仅管理员 │
│  下载学生表单      │   ✅   │   ✅   │    ✅    │  全部可用 │
│  管理表单          │   ❌   │   ⚠️  │    ✅    │  仅管理员 │
└──────────────────────────────────────────────────────────┘
```

**图例说明：**
- ✅ **允许访问**
- ❌ **禁止访问**
- ⚠️ **部分访问**（需要特定条件或授权）

### 权限验证机制

#### 1. 查询权限验证

```javascript
/**
 * 查询权限验证器
 * 设计师：内师智能体系统 (￣▽￣)ﾉ
 */
class QueryPermissionValidator {
  /**
   * 验证查询权限
   * @param {Object} user - 用户信息
   * @param {string} intent - 查询意图
   * @param {Object} params - 查询参数
   * @returns {Object} { allowed: boolean, message: string, filter: Object }
   */
  validate(user, intent, params = {}) {
    const { type: userType, id: userId } = user;

    // 个人资料查询
    if (intent === '个人资料' || intent === '个人信息') {
      // 学生和教师只能查询自己
      return {
        allowed: true,
        filter: { [userType + '_id']: userId },
        message: '只能查询自己的信息'
      };
    }

    // 成绩查询
    if (intent === '成绩查询' || intent === '成绩') {
      if (userType === 'student') {
        // 学生只能查自己的成绩
        return {
          allowed: true,
          filter: { student_id: userId },
          message: '只能查询自己的成绩'
        };
      } else if (userType === 'teacher') {
        // 教师可以查自己授课班级的学生成绩
        return {
          allowed: true,
          filter: this.getTeacherClassFilter(userId),
          message: '可查询授课班级的学生成绩'
        };
      } else if (userType === 'admin') {
        // 管理员可以查所有
        return {
          allowed: true,
          filter: {},
          message: '管理员权限'
        };
      }
    }

    // 课程查询
    if (intent === '课程查询' || intent === '课程') {
      if (userType === 'student') {
        // 学生只能查自己班级的课程
        return {
          allowed: true,
          filter: this.getStudentClassFilter(userId),
          message: '只能查询自己班级的课程'
        };
      } else if (userType === 'teacher') {
        // 教师可以查自己授课的课程
        return {
          allowed: true,
          filter: { teacher_id: userId },
          message: '只能查询自己授课的课程'
        };
      }
    }

    // 默认：管理员可以访问所有
    if (userType === 'admin') {
      return {
        allowed: true,
        filter: {},
        message: '管理员权限'
      };
    }

    // 其他情况：拒绝访问
    return {
      allowed: false,
      filter: {},
      message: '权限不足'
    };
  }

  /**
   * 获取教师的班级筛选条件
   */
  async getTeacherClassFilter(teacherId) {
    const { query } = require('../config/database');
    const classes = await query(`
      SELECT DISTINCT c.class_id
      FROM course_offerings co
      LEFT JOIN classes c ON co.class_id = c.class_id
      WHERE co.teacher_id = ?
    `, [teacherId]);

    return {
      class_id: classes.map(c => c.class_id)
    };
  }

  /**
   * 获取学生的班级筛选条件
   */
  async getStudentClassFilter(studentId) {
    const { query } = require('../config/database');
    const student = await query(`
      SELECT class_id FROM students WHERE student_id = ?
    `, [studentId]);

    return student.length > 0 ? { class_id: student[0].class_id } : {};
  }
}

module.exports = new QueryPermissionValidator();
```

#### 2. 下载权限验证

```javascript
/**
 * 下载权限验证器
 * 设计师：内师智能体系统 (￣▽￣)ﾉ
 */
class DownloadPermissionValidator {
  /**
   * 验证下载权限
   * @param {Object} user - 用户信息
   * @param {string} downloadType - 下载类型
   * @param {Object} params - 下载参数
   * @returns {Object} { allowed: boolean, message: string, scope: string }
   */
  validate(user, downloadType, params = {}) {
    const { type: userType, id: userId } = user;

    // 个人信息下载 - 所有用户都可以下载自己的
    if (downloadType === '个人信息') {
      return {
        allowed: true,
        scope: 'personal',
        filter: { [userType + '_id']: userId },
        message: '可以下载自己的个人信息'
      };
    }

    // 成绩单下载
    if (downloadType === '成绩单') {
      if (userType === 'student') {
        return {
          allowed: true,
          scope: 'personal',
          filter: { student_id: userId },
          message: '可以下载自己的成绩单'
        };
      } else if (userType === 'teacher') {
        return {
          allowed: true,
          scope: 'class',
          message: '可以下载授课班级的成绩单'
        };
      } else if (userType === 'admin') {
        return {
          allowed: true,
          scope: 'all',
          message: '管理员可以下载所有成绩单'
        };
      }
    }

    // 班级信息下载
    if (downloadType === '班级信息') {
      if (userType === 'student') {
        return {
          allowed: false,
          message: '学生不能下载班级信息，请联系老师或管理员'
        };
      } else if (userType === 'teacher') {
        return {
          allowed: true,
          scope: 'class',
          message: '可以下载授课班级的信息'
        };
      } else if (userType === 'admin') {
        return {
          allowed: true,
          scope: 'all',
          message: '管理员可以下载所有班级信息'
        };
      }
    }

    // 学院信息下载 - 仅管理员
    if (downloadType === '学院信息') {
      if (userType === 'admin') {
        return {
          allowed: true,
          scope: 'all',
          message: '管理员可以下载学院信息'
        };
      }
      return {
        allowed: false,
        message: '下载学院信息需要管理员权限'
      };
    }

    // 学生表单下载 - 所有用户都可以
    if (downloadType === '学生表单') {
      return {
        allowed: true,
        scope: 'public',
        message: '所有用户都可以下载学生表单'
      };
    }

    // 默认拒绝
    return {
      allowed: false,
      message: '未知的下载类型或权限不足'
    };
  }
}

module.exports = new DownloadPermissionValidator();
```

```javascript
/**
 * 查询权限验证器
 * 设计师：内师智能体系统 (￣▽￣)ﾉ
 * 功能：验证用户的查询权限，防止越权访问
 */

const { query } = require('../config/database');
const logger = require('../utils/logger');

class QueryPermissionValidator {
  /**
   * 验证查询权限
   * @param {Object} user - 用户信息 { type, id }
   * @param {string} intent - 查询意图
   * @param {Object} params - 查询参数
   * @returns {Promise<Object>} { allowed, message, filter }
   */
  async validate(user, intent, params = {}) {
    const { type: userType, id: userId } = user;

    logger.info('权限验证开始', { userType, userId, intent });

    // 个人资料查询 - 只能查自己
    if (intent === '个人资料' || intent === '个人信息') {
      return {
        allowed: true,
        filter: { [userType + '_id']: userId },
        message: '只能查询自己的信息'
      };
    }

    // 成绩查询
    if (intent === '成绩查询' || intent === '成绩') {
      if (userType === 'student') {
        // 学生只能查自己的成绩
        return {
          allowed: true,
          filter: { student_id: userId },
          message: '只能查询自己的成绩'
        };
      } else if (userType === 'teacher') {
        // 教师可以查自己授课班级的学生成绩
        const classIds = await this.getTeacherClassIds(userId);
        return {
          allowed: true,
          filter: classIds.length > 0 ? { student_id: classIds } : {},
          message: '可查询授课班级的学生成绩'
        };
      } else if (userType === 'admin') {
        // 管理员可以查所有
        return {
          allowed: true,
          filter: {},
          message: '管理员权限'
        };
      }
    }

    // 课程查询
    if (intent === '课程查询' || intent === '课程' || intent === '课表') {
      if (userType === 'student') {
        // 学生只能查自己班级的课程
        const classInfo = await this.getStudentClassInfo(userId);
        return {
          allowed: true,
          filter: classInfo ? { class_id: classInfo.class_id } : {},
          message: '只能查询自己班级的课程'
        };
      } else if (userType === 'teacher') {
        // 教师可以查自己授课的课程
        return {
          allowed: true,
          filter: { teacher_id: userId },
          message: '只能查询自己授课的课程'
        };
      } else if (userType === 'admin') {
        return {
          allowed: true,
          filter: {},
          message: '管理员权限'
        };
      }
    }

    // 管理办法查询 - 所有用户都可以
    if (intent === '管理办法') {
      return {
        allowed: true,
        filter: {},
        message: '可以查询管理办法'
      };
    }

    // 默认：管理员可以访问所有
    if (userType === 'admin') {
      return {
        allowed: true,
        filter: {},
        message: '管理员权限'
      };
    }

    // 其他情况：拒绝访问
    logger.warn('权限验证失败', { userType, userId, intent });
    return {
      allowed: false,
      filter: {},
      message: '权限不足，无法执行此查询'
    };
  }

  /**
   * 获取教师授课的班级ID列表
   * @param {number} teacherId - 教师ID
   * @returns {Promise<Array>} 班级ID列表
   */
  async getTeacherClassIds(teacherId) {
    try {
      const result = await query(`
        SELECT DISTINCT c.class_id
        FROM course_offerings co
        LEFT JOIN classes c ON co.class_id = c.class_id
        WHERE co.teacher_id = ? AND co.status = 1
      `, [teacherId]);

      return result.map(r => r.class_id);
    } catch (error) {
      logger.error('获取教师班级失败', { error: error.message, teacherId });
      return [];
    }
  }

  /**
   * 获取学生的班级信息
   * @param {number} studentId - 学生ID
   * @returns {Promise<Object>} 班级信息
   */
  async getStudentClassInfo(studentId) {
    try {
      const result = await query(`
        SELECT class_id FROM students WHERE student_id = ?
      `, [studentId]);

      return result.length > 0 ? result[0] : null;
    } catch (error) {
      logger.error('获取学生班级失败', { error: error.message, studentId });
      return null;
    }
  }
}

module.exports = new QueryPermissionValidator();
```

```javascript
/**
 * 上下文管理服务
 * 设计师：内师智能体系统 (￣▽￣)ﾉ
 * 功能：管理对话上下文，支持连续对话
 */

const { query } = require('../config/database');
const logger = require('../utils/logger');

class ContextService {
  constructor() {
    // 内存中的上下文缓存
    this.contextCache = new Map();

    // 定期清理过期上下文
    this.startCleanupTask();
  }

  /**
   * 获取会话上下文
   * @param {string} sessionId - 会话ID
   * @param {number} maxTurns - 最大轮数
   * @returns {Promise<Array>} 消息历史
   */
  async getContext(sessionId, maxTurns = 5) {
    try {
      // 1. 先从缓存获取
      const cached = this.contextCache.get(sessionId);
      if (cached && cached.messages.length > 0) {
        logger.info('从缓存获取上下文', { sessionId, messagesCount: cached.messages.length });
        return cached.messages.slice(-maxTurns * 2); // 每轮2条消息（user + assistant）
      }

      // 2. 从数据库获取
      const history = await query(`
        SELECT user_question, ai_answer, created_at
        FROM chat_history
        WHERE session_id = ?
        ORDER BY created_at DESC
        LIMIT ?
      `, [sessionId, maxTurns]);

      // 3. 构建消息数组（倒序，从早到晚）
      const messages = [];
      for (let i = history.length - 1; i >= 0; i--) {
        messages.push({
          role: 'user',
          content: history[i].user_question
        });
        messages.push({
          role: 'assistant',
          content: this.extractTextFromAnswer(history[i].ai_answer)
        });
      }

      // 4. 缓存到内存
      this.contextCache.set(sessionId, {
        messages,
        createdAt: Date.now()
      });

      logger.info('从数据库获取上下文', { sessionId, messagesCount: messages.length });
      return messages;

    } catch (error) {
      logger.error('获取上下文失败', { error: error.message, sessionId });
      return [];
    }
  }

  /**
   * 更新上下文缓存
   * @param {string} sessionId - 会话ID
   * @param {string} userQuestion - 用户问题
   * @param {string} aiAnswer - AI回答
   */
  updateContext(sessionId, userQuestion, aiAnswer) {
    try {
      const cached = this.contextCache.get(sessionId) || { messages: [] };

      cached.messages.push({
        role: 'user',
        content: userQuestion
      });

      cached.messages.push({
        role: 'assistant',
        content: this.extractTextFromAnswer(aiAnswer)
      });

      // 限制最大消息数（避免内存占用过大）
      if (cached.messages.length > 20) {
        cached.messages = cached.messages.slice(-20);
      }

      this.contextCache.set(sessionId, cached);
      logger.debug('更新上下文缓存', { sessionId, messagesCount: cached.messages.length });

    } catch (error) {
      logger.error('更新上下文失败', { error: error.message });
    }
  }

  /**
   * 清除会话上下文
   * @param {string} sessionId - 会话ID
   */
  clearContext(sessionId) {
    this.contextCache.delete(sessionId);
    logger.info('清除上下文', { sessionId });
  }

  /**
   * 从AI回答中提取纯文本（去除JSON等格式）
   * @param {string} answer - AI回答
   * @returns {string} 纯文本
   */
  extractTextFromAnswer(answer) {
    try {
      // 尝试解析JSON
      if (answer.trim().startsWith('{')) {
        const parsed = JSON.parse(answer);
        if (parsed.type === 'query' || parsed.type === 'profile' || parsed.type === 'grades') {
          // 对于查询结果，返回简短描述
          return parsed.message || parsed.data?.message || '查询完成';
        }
      }
      // 不是JSON，返回原文（截取前500字符）
      return answer.length > 500 ? answer.substring(0, 500) + '...' : answer;
    } catch (e) {
      // 解析失败，返回原文（截取前500字符）
      return answer.length > 500 ? answer.substring(0, 500) + '...' : answer;
    }
  }

  /**
   * 定期清理过期上下文
   */
  startCleanupTask() {
    // 每30分钟清理一次
    setInterval(() => {
      const now = Date.now();
      const maxAge = 1800000; // 30分钟

      for (const [sessionId, context] of this.contextCache.entries()) {
        if (now - context.createdAt > maxAge) {
          this.contextCache.delete(sessionId);
          logger.debug('清理过期上下文', { sessionId });
        }
      }

      logger.info('上下文清理完成', { remainingContexts: this.contextCache.size });
    }, 1800000); // 30分钟
  }

  /**
   * 获取上下文统计信息
   */
  getStats() {
    return {
      totalContexts: this.contextCache.size,
      contexts: Array.from(this.contextCache.entries()).map(([sessionId, context]) => ({
        sessionId,
        messagesCount: context.messages.length,
        createdAt: context.createdAt
      }))
    };
  }
}

module.exports = new ContextService();
```

## ⚠️ 注意事项

### 1. Token消耗

**问题：** 添加上下文会增加token消耗

**解决方案：**
- 限制历史轮数（建议3-5轮）
- 对历史消息进行压缩
- 定期清理过期上下文

### 2. 性能影响

**问题：** 每次都需要查询数据库获取历史

**解决方案：**
- 使用内存缓存（已实现）
- 设置合理的过期时间
- 考虑使用Redis缓存

### 3. 隐私问题

**问题：** 上下文可能包含敏感信息

**解决方案：**
- 不记录密码等敏感信息
- 定期清理历史记录
- 提供清除上下文的API

### 4. 数据库负载

**问题：** 频繁查询历史记录

**解决方案：**
- 优先使用缓存
- 减少数据库查询频率
- 考虑读写分离
