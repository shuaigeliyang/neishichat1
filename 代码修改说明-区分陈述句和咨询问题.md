### 错误现象

用户查询："我挂科了"

**前端日志：**
```
🎯 意图识别：文档问答（政策规定）
📚 调用RAG文档问答接口...
```

**系统回复：**
```
抱歉，根据学生手册，没有找到与您问题相关的信息...
建议您：
- 检查问题表述是否准确
- 咨询辅导员或相关部门
- 查阅学生手册纸质版
```

**预期行为：**
```
🎯 意图识别：普通聊天
💬 调用普通聊天接口（智谱AI）
```

**用户反馈：** "这个可以处理成普通回答吧"

---

## 🔍 根本原因分析

### 问题分析

**用户问题：** "我挂科了"

**问题类型：**
- ❌ 这不是在咨询政策
- ❌ 这不是在询问流程
- ✅ 这是一个**陈述句**，用户在陈述一个事实
- ✅ 用户希望得到安慰或建议，而不是查询文档

### 当前的错误逻辑

```javascript
const policyKeywords = [
  '怎么办', '怎么申请', ...  // 咨询性关键词
  '挂科', '不及格', ...      // 政策相关词汇
];

// ❌ 错误：只要包含"挂科"就识别为文档问答
if (question.includes('挂科')) {
  return 'document';
}
```

**问题：**
- "我挂科了"包含"挂科" → 被识别为文档问答
- 但"我挂科了"是陈述句，不是咨询问题！
- RAG检索找不到相关文档 → 返回"抱歉没有找到"
- 用户感到困惑和失望

### 正确的逻辑应该是

| 用户问题 | 问题类型 | 正确意图 | 理由 |
|---------|---------|---------|------|
| 我挂科了 | 陈述句 | 普通聊天 | 用户在陈述事实，需要安慰 |
| 挂科了怎么办 | 咨询问题 | 文档问答 | 用户在询问处理方法 |
| 挂科了会怎么样 | 咨询问题 | 文档问答 | 用户在询问后果 |
| 我不及格了 | 陈述句 | 普通聊天 | 用户在陈述事实 |
| 不及格怎么办 | 咨询问题 | 文档问答 | 用户在询问处理方法 |

### 修改后

```javascript
// 0.17 政策咨询关键词（优先级高于一般数据库查询）

// 🔒 重要：区分陈述句和咨询问题！
// 陈述句示例："我挂科了"、"我不及格"、"我违纪了" → 应该是普通聊天
// 咨询问题示例："挂科怎么办"、"挂科会怎么样" → 应该是文档问答

// 咨询性问题关键词（必须同时满足这些才识别为文档问答）
const inquiryKeywords = [
  '怎么办', '怎么处理', '如何处理', '怎么申请', '如何申请',
  '会怎么样', '有什么影响', '后果', '结果', '是什么', '有哪些',
  '怎么拿', '如何获得', '怎么获得', '怎么评', '如何评选',
  '怎么', '如何', '能不能', '可不可以', '是否', '可'
];

// 政策相关词汇（单独出现时不识别为文档问答）
const policyTopics = [
  // 学业问题
  '挂科', '不及格', '补考', '重修', '退学', '休学', '复学',
  '转专业', '选课', '退课', '旷课', '迟到', '早退', '请假',

  // 奖惩问题
  '奖励', '处分', '违纪', '作弊', '记过', '警告',

  // 奖学金相关
  '奖学金', '助学金', '助学贷款', '贫困生', '资助',

  // 其他常见政策咨询
  '规定', '制度', '政策', '流程', '条件', '要求', '标准',
  '办理', '手续', '材料'
];

// 检查是否同时包含咨询性关键词和政策相关词汇
const hasInquiryKeyword = inquiryKeywords.some(keyword =>
  question.toLowerCase().includes(keyword.toLowerCase())
);

const hasPolicyTopic = policyTopics.some(topic =>
  question.toLowerCase().includes(topic.toLowerCase())
);

// 🔒 特殊处理：排除陈述句模式
// 如果只是"我+事件+了"的模式（如"我挂科了"、"我不及格了"），不是咨询问题
const statementPattern = /^我.+了$/;
const isStatement = statementPattern.test(question.trim());

// ✅ 只有同时满足：咨询关键词 + 政策词汇 + 不是陈述句，才识别为文档问答
if (hasInquiryKeyword && hasPolicyTopic && !isStatement) {
  console.log('🎯 意图识别：文档问答（政策规定）');
  return 'document';
}
```

## 🎯 核心改进

### 1. 分离关键词类型

**咨询性关键词（inquiryKeywords）：**
- "怎么办"、"怎么"、"如何"
- "会怎么样"、"有什么影响"
- "能不能"、"可不可以"
- 这些词表示用户在**询问**

**政策相关词汇（policyTopics）：**
- "挂科"、"不及格"、"奖学金"
- "处分"、"违纪"、"作弊"
- 这些词表示用户在**谈论政策话题**

### 2. 同时满足两个条件

**修改前：**
```javascript
if (包含policyKeywords) {
  return 'document';
}
```

**修改后：**
```javascript
if (包含inquiryKeywords && 包含policyTopics && 不是陈述句) {
  return 'document';
}
```

**效果：**
- "我挂科了" → 包含"挂科"但**不包含咨询关键词** → 普通聊天 ✅
- "挂科怎么办" → 包含"挂科"+"怎么办" → 文档问答 ✅

### 3. 排除陈述句模式

**陈述句检测：**
```javascript
const statementPattern = /^我.+了$/;
const isStatement = statementPattern.test(question.trim());
```

**匹配的陈述句：**
- "我挂科了" ✅
- "我不及格了" ✅
- "我违纪了" ✅
- "我受处分了" ✅

**不匹配的咨询问题：**
- "挂科怎么办" ✅
- "我挂科了怎么办" ✅（包含"怎么办"）
- "挂科了会怎么样" ✅

## 💡 设计思路

### 为什么这样修改？

1. **区分陈述句和咨询问题**
   - 用户说"我挂科了"时，是在寻求安慰，不是查询文档
   - 用户说"挂科怎么办"时，是在查询政策，应该用文档问答

2. **同时满足两个条件**
   - 必须同时包含咨询性关键词和政策相关词汇
   - 避免单独的政策词汇被误识别为文档问答

3. **排除陈述句模式**
   - "我+事件+了"是典型的陈述句模式
   - 这种模式应该让智谱AI来处理，提供情感支持

## 🎯 总结

### 修改内容

在`intentDetector.js`的政策咨询检测逻辑中：
1. ✅ 将关键词分为咨询性关键词和政策相关词汇
2. ✅ 要求同时满足两个条件才识别为文档问答
3. ✅ 添加陈述句检测，排除"我+事件+了"模式

### 解决的问题

- ✅ "我挂科了"现在被识别为普通聊天
- ✅ 用户得到情感支持，而不是冷冰冰的"没找到"
- ✅ "挂科怎么办"仍然是文档问答，不受影响
- ✅ 所有陈述句都能正确降级到普通聊天

### 技术要点

- **双重条件检测**：咨询关键词 + 政策词汇
- **陈述句模式匹配**：正则表达式`/^我.+了$/`
- **优先级保持**：不影响其他意图识别
