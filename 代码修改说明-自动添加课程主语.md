```javascript
function normalizeQuestion(question) {
    let normalized = question;

    // 🔒 特殊处理1：当问题关于"挂科"或"不及格"时，如果缺少主语，自动添加"学生"
    if (
        (question.includes('挂科') || question.includes('不及格')) &&
        !question.includes('学生') &&
        !question.includes('我的') &&
        (question.includes('怎么办') || question.includes('会怎么样') ||
         question.includes('怎么处理') || question.includes('如何处理'))
    ) {
        console.log('📝 检测到缺少主语，自动添加"学生"主语');
        console.log(`   原始问题：${question}`);

        // 在"挂科"前添加"学生"
        normalized = normalized.replace(/挂科/g, '学生挂科');

        console.log(`   修正后问题：${normalized}`);
        console.log('   💡 这样可以更好地匹配到学生手册中的文档');
    }

    // 🔒 特殊处理2：当问题关于"重修"、"补考"时，如果缺少主语，自动添加"课程"
    // 文档中是"课程重修管理办法"、"课程补考管理办法"
    if (
        (question.includes('重修') || question.includes('补考')) &&
        !question.includes('课程') &&
        !question.includes('学生') &&
        !question.includes('我的') &&
        (question.includes('可不可以') || question.includes('能不能') ||
         question.includes('怎么办') || question.includes('怎么') || question.includes('如何'))
    ) {
        console.log('📝 检测到缺少主语，自动添加"课程"主语');
        console.log(`   原始问题：${question}`);

        // 在"重修"或"补考"前添加"课程"
        normalized = normalized.replace(/重修/g, '课程重修');
        normalized = normalized.replace(/补考/g, '课程补考');

        console.log(`   修正后问题：${normalized}`);
        console.log('   💡 这样可以更好地匹配到课程重修/补考相关文档');
    }

    // 替换所有同义词
    for (const [colloquial, formal] of Object.entries(SYNONYM_MAP)) {
        const regex = new RegExp(colloquial, 'g');
        normalized = normalized.replace(regex, formal);
    }

    // 如果问题发生了变化，记录日志
    if (normalized !== question) {
        console.log(`📝 问题标准化：`);
        console.log(`   原始：${question}`);
        console.log(`   标准化：${normalized}`);
    }

    return normalized;
}
```

## 🎯 触发条件

### 自动添加"课程"主语的条件

当问题**同时满足**以下所有条件时，会自动添加"课程"主语：

1. ✅ 包含"重修"或"补考"
2. ✅ 不包含"课程"（避免重复）
3. ✅ 不包含"学生"（避免冲突）
4. ✅ 不包含"我的"（避免与"我的课程"重复）
5. ✅ 包含咨询性关键词：
   - "可不可以"、"能不能"
   - "怎么办"、"怎么"、"如何"

### 触发示例

| 用户问题 | 是否触发 | 修正后问题 | 说明 |
|---------|---------|-----------|------|
| 可不可以重修 | ✅ 触发 | 课程可不可以重修 | 满足所有条件 |
| 能不能补考 | ✅ 触发 | 课程能不能补考 | 满足所有条件 |
| 重修怎么办 | ✅ 触发 | 课程重修怎么办 | 满足所有条件 |
| 补考怎么申请 | ✅ 触发 | 课程补考怎么申请 | 满足所有条件 |
| 课程可不可以重修 | ❌ 不触发 | （不变） | 已有"课程" |
| 学生可不可以重修 | ❌ 不触发 | （不变） | 已有"学生" |
| 我的课程重修 | ❌ 不触发 | （不变） | 已有"我的" |
| 重修和补考的区别 | ❌ 不触发 | （不变） | 没有咨询性关键词 |

## 💡 设计思路

### 为什么添加"课程"而不是"学生"？

**原因分析：**

1. **文档内容的特殊性**
   - 文档标题："内江师范学院**课程**重修管理办法"
   - 文档标题："内江师范学院**课程**补考管理办法"
   - 文档中更多提到的是"课程"而不是"学生"

2. **语义准确性**
   - "课程重修"是更准确的表达
   - 重修的对象是"课程"，不是"学生"
   - "学生重修"可能被理解为"学生重读（年级）"

3. **向量匹配效果**
   - "课程重修"与文档标题"课程重修管理办法"匹配度更高
   - "学生重修"与文档的向量距离较远

### 为什么选择这些触发条件？

**4个必须条件：**

1. **包含"重修"或"补考"**
   - 确保是重修/补考相关的问题

2. **不包含"课程"**
   - 避免重复添加

3. **不包含"学生"和"我的"**
   - 避免与"学生重修"或"我的课程重修"冲突

4. **包含咨询性关键词**
   - "可不可以"、"能不能" → 询问是否允许
   - "怎么办"、"怎么"、"如何" → 询问流程
   - 确保是咨询类问题，而不是陈述句
