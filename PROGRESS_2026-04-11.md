## 一、已完成的核心修改

### 1. 多文档RAG服务重构 ✅
**文件：** `backend/services/multiDocumentRagService.js`

**设计理念（自包含架构）：**
- 直接从 `unified_index.json` 读取chunks和embeddings
- 不依赖外部缓存文件
- 内存中维护索引数据

**关键参数：**
- 置信度阈值：`0.35`（低于此值不调用LLM）
- 默认TopK：`5`
- 最小分数：`0.2`

**检索流程：**
1. 生成问题向量（embedding-3模型）
2. 计算余弦相似度
3. 按文档分组返回结果
4. 严格基于检索内容生成答案（禁止编造）

### 问题：429 Rate Limit（账户级限流）
**现象：** 智谱AI返回429 Too Many Requests错误

**原因分析：**
- 账户级别的QPS限制（非Key级别）
- 测试期间频繁调用API导致触发限流
- 新Key已确认正确加载（Authorization头验证通过）

**预计恢复时间：**
- QPS限流：几秒到几分钟后自动恢复
- 日额度限制：可能需要等待或充值

**代码已有保护措施：**
- `embeddingService.js`：请求队列 + 指数退避重试（最多3次）
- `multiDocumentRagService.js`：置信度阈值过滤，避免无意义调用

## 五、架构说明

```
用户问题
   ↓
意图识别（intentDetector.js）
   ↓
┌─────────────────────────────────────┐
│ database  → 数据库查询               │
│ document  → RAG文档问答              │
│ form_list → 表单列表查询             │
│ form_generate → 表单生成             │
│ chat     → 智谱AI对话               │
└─────────────────────────────────────┘
   ↓
RAG服务（multiDocumentRagService.js）
   ↓
┌─────────────────────────────────────┐
│ 1. generateEmbedding() → 生成向量    │
│ 2. search() → 余弦相似度检索         │
│ 3. 置信度检查（<0.35则返回空）        │
│ 4. generateAnswer() → LLM生成答案    │
└─────────────────────────────────────┘
   ↓
返回答案 + 来源 + 置信度
```
