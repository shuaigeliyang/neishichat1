## 📋 系统概述

本系统实现了对多个政策文档的统一管理，支持：
- ✅ 多文档独立管理（新增、替换、删除）
- ✅ 统一索引聚合检索
- ✅ 智能问答自动标注来源
- ✅ 增量更新，无需重建整个索引

## 🚀 快速开始

### 1. 迁移现有数据

如果你有现有的学生手册数据，运行迁移脚本：

```bash
cd backend
node scripts/migrate-to-registry.js
```

这将：
- 创建 `文档库/registry.json`
- 创建 `文档库/indexes/unified_index.json`
- 将现有学生手册数据迁移到新结构

### 2. 测试系统

运行测试脚本验证系统功能：

```bash
# 基础测试（不需要API调用）
node scripts/test-multi-doc-system.js

# 完整测试（包含RAG问答，需要API）
node scripts/test-multi-doc-system.js --rag
```

### 3. 启动服务器

```bash
cd backend
npm run dev
```

服务器将在 `http://localhost:3000` 启动

## 📡 API接口

### 文档管理

#### 1. 获取所有文档

```http
GET /api/documents
```

**响应示例：**
```json
{
  "success": true,
  "data": {
    "documents": [...],
    "statistics": {
      "totalDocuments": 2,
      "indexedDocuments": 1,
      "pendingDocuments": 1
    }
  }
}
```

#### 2. 上传新文档

```http
POST /api/documents/upload
Content-Type: multipart/form-data
```

**参数：**
- `file`: 文件（必填，支持 .docx, .doc, .pdf）
- `name`: 文档名称（可选）
- `description`: 文档描述（可选）
- `tags`: 标签，逗号分隔（可选）

**示例（curl）：**
```bash
curl -X POST http://localhost:3000/api/documents/upload \
  -F "file=@奖学金评定办法.docx" \
  -F "name=奖学金评定办法" \
  -F "description=学生奖学金评定相关办法" \
  -F "tags=奖学金,评定,奖励"
```

**响应示例：**
```json
{
  "success": true,
  "data": {
    "documentId": "policy_xxx",
    "name": "奖学金评定办法",
    "status": "pending",
    "directory": "文档提取/奖学金评定办法"
  },
  "message": "文档上传成功，请开始处理"
}
```

#### 3. 处理文档

```http
POST /api/documents/:documentId/process
```

**说明：** 启动文档处理流程（提取→分块→向量化→索引）

**响应示例：**
```json
{
  "success": true,
  "data": {
    "documentId": "policy_xxx",
    "status": "processing"
  },
  "message": "文档处理已启动"
}
```

#### 4. 获取文档状态

```http
GET /api/documents/:documentId/status
```

**响应示例：**
```json
{
  "success": true,
  "data": {
    "documentId": "policy_xxx",
    "name": "奖学金评定办法",
    "status": "indexed",
    "statistics": {
      "totalPages": 50,
      "totalChunks": 200,
      "indexedChunks": 200
    }
  }
}
```

#### 5. 替换文档

```http
POST /api/documents/:documentId/replace
Content-Type: multipart/form-data
```

**参数：**
- `file`: 新文件（必填）

#### 6. 删除文档

```http
DELETE /api/documents/:documentId
```

### 索引管理

#### 1. 获取统一索引状态

```http
GET /api/index/status
```

**响应示例：**
```json
{
  "success": true,
  "data": {
    "totalDocuments": 2,
    "totalChunks": 1040,
    "documents": [
      {
        "documentId": "policy_001",
        "name": "2025年本科学生手册-定",
        "indexedAt": "2026-04-10T12:00:00.000Z"
      }
    ]
  }
}
```

## 🔧 开发指南

### 添加新文档的完整流程

```bash
# 1. 准备文档文件
# 将文档放到 相关文档/ 目录

# 2. 上传文档（通过API或直接操作）
curl -X POST http://localhost:3000/api/documents/upload \
  -F "file=@相关文档/新政策.docx" \
  -F "name=新政策名称"

# 3. 处理文档（提取内容）
# 如果是.docx文件，需要先提取内容：
node backend/scripts/extract-docx.js 文档提取/新政策名称/新政策.docx

# 4. 开始处理（分块→向量化→索引）
curl -X POST http://localhost:3000/api/documents/{documentId}/process

# 5. 查看状态
curl http://localhost:3000/api/documents/{documentId}/status
```

### 使用多文档RAG服务

```javascript
const MultiDocumentRAGService = require('./backend/services/multiDocumentRagService');

const ragService = new MultiDocumentRAGService(apiKey, {
    embeddingMode: 'api'  // 或 'local'
});

await ragService.initialize();

const result = await ragService.ask('奖学金怎么评？', {
    topK: 5,
    minScore: 0.5
});

console.log(result.answer);
console.log('来源:', result.sources);
```

## 🎯 常见问题

### Q: 如何添加新的政策文档？

A: 有两种方式：
1. **通过API上传**（推荐）：使用 `POST /api/documents/upload` 接口
2. **手动操作**：将文件放到 `相关文档/` 目录，然后手动创建文档目录

### Q: 文档处理需要多长时间？

A: 取决于文档大小：
- 50页文档：约2-5分钟
- 100页文档：约5-10分钟
- 200页文档：约10-20分钟

### Q: 可以同时处理多个文档吗？

A: 可以。每个文档处理是独立的，可以同时启动多个文档的处理流程。

### Q: 如何知道文档处理完成了？

A: 通过 `GET /api/documents/:documentId/status` 查询状态，当 `status` 为 `indexed` 时表示处理完成。

### Q: 删除文档会影响其他文档吗？

A: 不会。每个文档独立管理，删除一个文档只会从统一索引中移除该文档的chunks，不影响其他文档。

## 📊 性能优化建议

### 1. 使用本地Embedding

对于大量文档，建议使用本地Embedding模型（完全免费，无API调用限制）：

```bash
# 在 .env 文件中设置
EMBEDDING_MODE=local
```

### 2. 缓存管理

向量缓存会自动保存到 `embedding_cache.json`，重启服务时会自动加载，避免重复生成向量。

### 3. 批量处理

如果需要添加多个文档，建议逐个上传和处理，避免同时处理过多文档导致资源占用过高。
