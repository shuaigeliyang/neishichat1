## 📋 后续步骤

### 1. 清理 TypeScript 文件

运行提供的清理脚本：
```bash

```

或手动删除：
- 后端：删除 `backend/src/**/*.ts`
- 前端：删除 `frontend/src/**/*.{ts,tsx}`
- 配置：删除 `tsconfig.json`, `vite.config.ts`

### 2. 重新安装依赖

```bash
# 后端
cd backend
bun install

# 前端
cd ../frontend
bun install
```

### 3. 启动项目

```bash
# 后端 (端口 3005)
cd backend
bun run dev

# 前端 (端口 5176)
cd frontend
bun run dev
```

### 4. 生产部署

**前端构建**：
```bash
cd frontend
bun run build
```

**后端启动**：
```bash
cd backend
bun run start
```

## 🔧 配置说明

### API 地址配置

前端使用 `buildApiUrl()` 函数统一管理 API 地址：

```javascript
import { buildApiUrl } from '@/config/api.js'

// 开发环境：自动使用 Vite 代理
fetch(buildApiUrl('/api/documents'))

// 生产环境：使用环境变量配置的完整 URL
// VITE_API_BASE_URL=https://api.example.com
```

### Vite 代理配置

`vite.config.js` 已配置代理：
```javascript
proxy: {
  '/api': {
    target: 'http://localhost:3005',
    changeOrigin: true,
  }
}
```

## ⚠️ 注意事项

1. **环境变量**：部署时记得设置 `VITE_API_BASE_URL`
2. **CORS**：确保后端 CORS 配置允许前端域名
3. **端口**：前端默认 5176，后端默认 3005
