# 教育系统智能体改造实施计划

## 项目概述

根据需求文档，需要对内江师范学院教育系统智能体进行三项重要改造：

### 核心目标
1. 创建圆形浮窗界面，支持未登录/登录双模式
2. 开发后台管理系统，支持动态更新智能体数据
3. 优化学生手册政策问答的输出质量

---

## 任务一：创建内江师范学院智能体圆形浮窗界面

### 1.1 设计方案

#### 浮窗界面结构
```
index-float.html (独立入口页面)
├── circular-icon-btn (圆形图标按钮)
│   ├── 学校Logo/智能体头像
│   └── 点击展开动画
├── chat-modal (对话框浮窗)
│   ├── 未登录模式（默认：手机尺寸，约375x667px）
│   │   ├── 基本问答功能
│   │   ├── 学校介绍
│   │   └── 登录引导
│   ├── 全屏模式（可选切换）
│   └── 登录后模式（完整功能）
└── 样式和动画
```

#### 功能分层设计

**未登录状态（访客模式）：**
- ✅ 学校基本介绍
- ✅ 礼貌性对话
- ✅ 简单问题回答
- ❌ 表单下载
- ❌ 数据库查询
- ❌ 政策查询
- 💡 引导登录以使用完整功能

**登录后状态（完整模式）：**
- ✅ 所有原有功能
- ✅ 表单下载
- ✅ 数据库查询
- ✅ 政策查询
- ✅ 权限控制（学生/教师/管理员）

### 1.2 技术实现要点

#### 前端实现
1. 创建 `frontend/public/index-float.html` 作为独立入口
2. 创建 `frontend/src/components/FloatChat.jsx` 浮窗聊天组件
3. 创建 `frontend/src/components/CircularIcon.jsx` 圆形图标组件
4. 创建 `frontend/src/pages/GuestChat.jsx` 访客聊天页面
5. 修改 `frontend/src/App.jsx` 添加浮窗路由

#### 后端实现
1. 创建 `backend/src/routes/guest.js` 访客API路由
2. 创建 `backend/src/services/guestChat.js` 访客聊天服务
3. 实现基本的问答逻辑（硬编码学校介绍等）
4. 添加登录检测和权限提示中间件

#### API接口设计

### 2.2 核心功能模块

#### 2.2.1 数据库信息管理
**功能：**
- 学生信息的增删改查
- 教师信息的增删改查
- 课程和成绩信息的可视化编辑
- 批量导入/导出功能

**API接口：**
```javascript
GET    /api/admin/data/students - 获取学生列表
POST   /api/admin/data/students - 添加学生
PUT    /api/admin/data/students/:id - 更新学生信息
DELETE /api/admin/data/students/:id - 删除学生
// 同理处理教师、课程、成绩等
```

#### 2.2.2 政策文件管理
**功能：**
- 上传新的政策文件（PDF/Word/文本）
- 文件内容自动解析和提取
- 更新RAG知识库索引
- 政策版本管理

**API接口：**
```javascript
POST   /api/admin/policy/upload - 上传政策文件
GET    /api/admin/policy/list - 获取政策列表
DELETE /api/admin/policy/:id - 删除政策
POST   /api/admin/policy/reindex - 重建知识库索引
```

#### 2.2.3 学生手册管理
**功能：**
- 上传新版学生手册
- 手册内容自动分页和提取
- 更新检索索引
- 手册版本对比

**API接口：**
```javascript
POST   /api/admin/handbook/upload - 上传学生手册
GET    /api/admin/handbook/versions - 获取版本列表
POST   /api/admin/handbook/rebuild - 重建手册索引
```

## 任务三：优化学生手册政策返回内容的格式

### 3.1 问题分析

#### 当前问题
1. **噪音问题：** 多余的标点符号和特殊字符
2. **格式问题：** 文本格式不统一，影响阅读体验
3. **内容质量：** 提取的内容包含无关信息

### 3.2 优化方案

#### 3.2.1 文本清理策略
```javascript
// 清理规则
1. 标准化标点符号：统一使用中文标点
2. 删除多余空白：连续空格、换行
3. 移除特殊字符：控制字符、乱码
4. 修正格式错误：错误的中英文混合
5. 段落重组：逻辑分块
```

#### 3.2.2 实现位置
修改 `backend/src/services/ragService.js` 或相关文档处理服务

#### 3.2.3 优化流程
```
原始内容
  ↓
文本清理（去除噪音）
  ↓
格式标准化
  ↓
内容验证
  ↓
输出优化后的内容
```

### 3.3 实施步骤
1. ✅ 分析当前RAG返回内容的格式问题
2. ✅ 设计文本清理和格式化规则
3. ✅ 实现内容优化服务
4. ✅ 集成到RAG问答流程
5. ✅ 测试优化效果
6. ✅ 对比优化前后的质量

## 风险评估

### 技术风险
1. **RAG知识库更新复杂度：** 需要确保更新不影响现有功能
   - 缓解措施：实现版本回滚机制

2. **浮窗兼容性：** 不同浏览器的浮窗实现可能有差异
   - 缓解措施：使用成熟的UI组件库

3. **数据一致性：** 后台更新数据时可能出现冲突
   - 缓解措施：实现乐观锁和事务处理

### 时间风险
- 时间紧迫，可能需要并行开发
- 优先级：浮窗界面 > 内容优化 > 后台系统
