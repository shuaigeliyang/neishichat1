## 🎯 问题根源

### 核心问题

**向量检索失败！** 虽然document_chunks.json中有"不及格"相关内容，但是：

1. ❌ 向量索引可能没有正确建立
2. ❌ 或者检索算法有问题
3. ❌ 或者minScore阈值太高（0.3）

### 问题分析

#### 可能性1：向量索引问题

从测试结果看：
```
✓ 索引已加载，共4385个文档块
✓ 索引时间：2026-03-25T03:13:32.292Z
```

索引是存在的，但是：
- 可能是旧的索引（不包含"不及格"的向量）
- 或者向量质量不好

#### 可能性2：检索阈值太高

当前使用：
```javascript
minScore: options.minScore || 0.3  // 默认0.3
```

如果"不及格"和"挂科"的语义相似度低于0.3，就不会被检索到。

#### 可能性3：问题标准化不匹配

问题标准化：
```
原始问题："挂科了会怎么样"
标准化问题："学生不及格了会后果"
```

但是文档中的内容是：
```
"成绩 不及格 <60 F (0.00) 0.00"
```

可能语义差异太大，导致向量距离太远。

---

## 🛠️ 解决方案

### 方案1：重建向量索引（推荐）✨

**步骤：**

1. 删除旧的向量索引
2. 重新建立索引
3. 确保所有文档块都被正确向量化

**实施：**

```bash
cd backend

# 删除旧索引
rm -f retrieval_index.json embedding_cache.json

# 重启后端服务
npm start
```

**预期效果：**
- RAG服务初始化时会重新建立索引
- 所有文档块（包括"不及格"相关内容）都会被正确向量化
- 检索时应该能找到相关文档

---

### 方案2：降低检索阈值

**修改：** `backend/src/routes/rag.js`

```javascript
// 修改前
const result = await ragService.answer(normalizedQuestion, {
    ...options,
    minScore: options.minScore || 0.3  // 默认0.3
});

// 修改后
const result = await ragService.answer(normalizedQuestion, {
    ...options,
    minScore: options.minScore || 0.1  // 降低到0.1
});
```

**效果：**
- 降低阈值会增加检索到的文档数量
- 但也可能检索到不相关的文档
- 需要权衡准确率和召回率

---

### 方案3：改进问题标准化

**当前标准化：**
```javascript
"挂科了会怎么样" → "学生不及格了会后果"
```

**问题：**
- 添加了"学生"这个词
- "会怎么样" → "会后果"（不太自然）

**改进：**
```javascript
// 更精确的标准化
"挂科了会怎么样" → "不及格会有什么后果"
// 或
"挂科了会怎么样" → "成绩不及格会怎么样"
```

---

### 方案4：添加关键词匹配作为补充

如果向量检索失败，使用关键词匹配作为后备：

```javascript
// 1. 尝试向量检索
let retrievedDocs = await this.retrievalEngine.retrieve(question, {
    topK: options.topK || this.options.topK,
    minScore: options.minScore || this.options.minScore,
    useReranking: options.useReranking !== false
});

// 2. 如果向量检索失败，使用关键词匹配
if (retrievedDocs.length === 0) {
    console.log('⚠️ 向量检索失败，尝试关键词匹配...');
    retrievedDocs = await this.retrievalEngine.keywordSearch(question, {
        topK: options.topK || this.options.topK
    });
}
```
