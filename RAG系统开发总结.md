## 🏗️ 系统架构

### 整体流程

```
学生提问
    ↓
【问题向量化】
    ↓
【语义检索】→ 找到Top-5相关文档
    ↓
【上下文构建】→ 组织文档片段
    ↓
【答案生成】→ 智谱AI生成回答

```

## 💡 技术亮点

### 1. 智能分块策略

```javascript
// 优先级：章节 > 段落 > 固定长度
1. 按章节分块（识别"第X章"、"一、"等标题）
2. 章节内按段落分块（500-1000字）
3. 添加上下文窗口（前后各100字）
4. 标注元数据（页码、章节、类型）
```

### 2. 混合检索算法

```javascript
// 结合语义相似度和关键词匹配
finalScore = semanticScore * 0.7 + keywordScore * 0.3
```

### 3. 质量检查机制

```javascript
// 检查回答质量
- 最高相关度是否达标
- 参考文档数量是否充足
- 是否包含"不知道"等关键词
```

### 4. 缓存优化

```javascript
// 向量缓存策略
- 内存Map：快速访问
- 文件持久化：重启后恢复
- 避免重复计算：节省成本
```

## 🎓 经验总结

### 开发经验

1. **文档处理很重要**
   - PDF提取质量直接影响后续效果
   - 智能分块比固定长度效果更好
   - 上下文窗口能显著提升准确性

2. **向量化是关键**
   - 选择合适的embedding模型很重要
   - 缓存机制能显著提升性能
   - 批量处理能节省API调用成本

3. **检索算法需要优化**
   - 纯语义搜索可能不够准确
   - 结合关键词匹配效果更好
   - 重排序能提升Top-K准确性

4. **提示词设计很重要**
   - 明确要求基于文档回答
   - 要求添加引用标注
   - 不知道时要明确告知

### 遇到的问题

1. **Python环境问题**
   - 多个Python环境冲突
   - 解决：使用指定路径的Python

2. **编码问题**
   - Windows控制台GBK编码
   - 解决：设置UTF-8编码输出

3. **文件路径问题**
   - Windows路径分隔符
   - 解决：使用正斜杠或双引号
