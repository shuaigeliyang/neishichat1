## 🔍 问题分析

### 数据库设计

**学生选课信息存储位置：**
- ❌ **没有** enrollments（选课表）
- ✅ **有** grades（成绩表），通过 offering_id 关联课程开课信息

**陈丽的选课情况：**
- grades表中有16条成绩记录
- 每条记录对应一门课（通过 offering_id 关联）

### AI生成的错误SQL

**错误的SQL：**
```sql
SELECT * FROM course_offerings WHERE teacher_id = 1
```

**问题：**
1. ❌ 查询的是教师的授课表（course_offerings）
2. ❌ 使用了 teacher_id = 1（赵燕老师）
3. ❌ 没有从 grades 表开始查询

### 正确的查询方式

**正确的SQL：**
```sql
SELECT
  c.course_name AS '课程名称',
  c.course_code AS '课程代码',
  co.semester AS '学期',
  co.schedule AS '上课时间',
  co.classroom AS '教室',
  t.name AS '授课教师',
  cl.class_name AS '班级'
FROM grades g
LEFT JOIN course_offerings co ON g.offering_id = co.offering_id
LEFT JOIN courses c ON co.course_id = c.course_id
LEFT JOIN teachers t ON co.teacher_id = t.teacher_id
LEFT JOIN classes cl ON co.class_id = cl.class_id
WHERE g.student_id = 1
ORDER BY co.semester DESC, c.course_name
```

**查询流程：**
1. ✅ 从 grades 表开始（WHERE student_id = 1）
2. ✅ 通过 offering_id 关联 course_offerings 表
3. ✅ 通过 course_id 关联 courses 表
4. ✅ 通过 teacher_id 关联 teachers 表
5. ✅ 返回学生的选课信息

## 🔧 技术细节

### 学生查询课程表的正确流程

**表结构：**
```
grades（成绩表）
  ├─ student_id（学生ID）
  ├─ offering_id（课程开课ID）
  └─ total_score（总评成绩）

course_offerings（课程开课表）
  ├─ offering_id（开课ID）
  ├─ course_id（课程ID）
  ├─ teacher_id（教师ID）
  ├─ semester（学期）
  ├─ schedule（上课时间）
  └─ classroom（教室）

courses（课程表）
  ├─ course_id（课程ID）
  ├─ course_name（课程名称）
  └─ course_code（课程代码）

teachers（教师表）
  ├─ teacher_id（教师ID）
  └─ name（教师姓名）

classes（班级表）
  ├─ class_id（班级ID）
  └─ class_name（班级名称）
```

**JOIN顺序：**
1. FROM grades g（主表，学生成绩）
2. LEFT JOIN course_offerings co（课程开课信息）
3. LEFT JOIN courses c（课程基本信息）
4. LEFT JOIN teachers t（授课教师）
5. LEFT JOIN classes cl（班级信息）

**WHERE条件：**
- `WHERE g.student_id = ${userContext.id}`（当前登录学生的ID）

**ORDER BY：**
- `ORDER BY co.semester DESC, c.course_name`（按学期降序、课程名称排序）

## 💡 关键规则

### 学生查询课程表的黄金法则

1. **必须从 grades 表开始**
   - ❌ 错误：`FROM course_offerings WHERE teacher_id = ?`
   - ✅ 正确：`FROM grades g WHERE g.student_id = ?`

2. **通过 offering_id 关联**
   - ❌ 错误：直接查询 course_offerings
   - ✅ 正确：`LEFT JOIN course_offerings co ON g.offering_id = co.offering_id`

3. **使用 student_id 作为WHERE条件**
   - ❌ 错误：`WHERE teacher_id = ?`
   - ✅ 正确：`WHERE g.student_id = ?`

4. **LEFT JOIN 确保数据完整性**
   - ✅ 使用 LEFT JOIN 而不是 INNER JOIN
   - ✅ 避免因缺少关联记录导致数据丢失
