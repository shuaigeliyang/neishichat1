## 📋 问题描述

### 用户需求

用户查询："我的挂科记录"

**当前行为：**
- AI生成错误的SQL，查询students表
- 返回个人信息（姓名、学号、性别等）
- ❌ 教师和管理员也能查询（隐私泄露）
- ❌ 没有处理"没有挂科"的情况

**期望行为：**
- ✅ 只有学生可以查询自己的挂科记录
- ✅ 教师和管理员无权查询（返回友好提示）
- ✅ 如果有挂科，列出挂科课程
- ✅ 如果没有挂科，切换到普通聊天，表扬学生"你真棒"

---

## 🔍 根本原因分析

### 问题1：权限控制缺失

**当前逻辑：**
```javascript
// AI生成SQL
SELECT * FROM students WHERE student_id = ?
```

**问题：**
- 教师查询"我的挂科记录" → AI可能查询students表 → 返回其他学生的信息
- 管理员查询"我的挂科记录" → AI可能查询grades表 → 看到所有学生的挂科记录
- 没有明确的权限控制

### 问题2：SQL生成不准确

**AI生成的SQL：**
```javascript
SELECT * FROM students WHERE student_id = 1
```

**应该是：**
```javascript
SELECT * FROM grades WHERE student_id = 1 AND (total_score < 60 OR is_retake = 1)
```

### 问题3：缺少"没有挂科"的处理

**当前：**
- 查询结果为空 → 返回"没有找到相关信息"
- 学生感到困惑

**应该：**
- 查询结果为空 → 切换到普通聊天 → 表扬学生"你真棒"

**位置：** `handleDatabaseQuery`函数

```javascript
const handleDatabaseQuery = async (question) => {
  try {
    const response = await axios.post(
      '/api/intelligent/query',
      { message: question, sessionId },
      { headers: { Authorization: `Bearer ${localStorage.getItem('token')}` } }
    );

    if (response.data.success) {
      // 🔒 特殊处理1：检查是否需要切换到普通聊天（表扬消息）
      if (response.data.meta?.switchToChat) {
        console.log('🎉 后端返回switchToChat标记，显示表扬消息');

        setMessages(prev => [...prev, {
          role: 'assistant',
          content: response.data.answer,
          timestamp: new Date().toISOString()
        }]);

        return;
      }

      // 🔒 特殊处理2：检查是否是权限拒绝或特殊消息
      if (response.data.meta?.type === 'permission_denied' || response.data.meta?.type === 'no_failed_grades') {
        console.log('📋 后端返回特殊消息:', response.data.meta?.type);

        setMessages(prev => [...prev, {
          role: 'assistant',
          content: response.data.answer,
          timestamp: new Date().toISOString()
        }]);

        return;
      }

      // 🔒 特殊处理3：如果answer不是JSON，直接显示
      const aiContent = response.data.data.answer;
      if (!aiContent || !aiContent.trim().startsWith('{')) {
        console.log('📝 answer不是JSON格式，直接显示');

        setMessages(prev => [...prev, {
          role: 'assistant',
          content: aiContent || response.data.answer,
          timestamp: new Date().toISOString()
        }]);

        return;
      }

      // 正常的智能查询结果处理
      const parsed = JSON.parse(aiContent);
      if (parsed.type === 'intelligent_query') {
        setMessages(prev => [...prev, {
          role: 'assistant',
          content: parsed.message,
          isIntelligentQuery: true,
          queryData: parsed.data,
          rowCount: parsed.rowCount,
          downloadUrl: parsed.downloadUrl,
          responseType: parsed.responseType,
          explanation: parsed.explanation,
          suggestions: parsed.suggestions || []
        }]);
        return;
      }
    }

    // 如果不是智能查询结果，回退到普通聊天
    await handleChatQuery(question);

  } catch (error) {
    // 错误处理...
  }
};
```

## 📊 SQL查询逻辑

### 挂科记录的判断标准

```sql
WHERE g.student_id = ${userId}
  AND (g.total_score < 60 OR g.is_retake = 1)
```

**说明：**
- `total_score < 60`：总评成绩低于60分
- `is_retake = 1`：标记为重修的课程

### 返回的字段

```sql
SELECT
  g.grade_id AS '成绩ID',
  c.course_name AS '课程名称',
  c.course_code AS '课程代码',
  g.total_score AS '总评成绩',
  co.semester AS '学期',
  g.is_retake AS '是否重修',
  t.name AS '授课教师'
FROM grades g
LEFT JOIN course_offerings co ON g.offering_id = co.offering_id
LEFT JOIN courses c ON co.course_id = c.course_id
LEFT JOIN teachers t ON co.teacher_id = t.teacher_id
WHERE g.student_id = ${userId}
  AND (g.total_score < 60 OR g.is_retake = 1)
ORDER BY co.semester DESC, c.course_name
```

## 💡 设计思路

### 1. 为什么不在AI生成SQL时处理？

**原因：**
- AI生成的SQL可能不准确
- 可能查询students表而不是grades表
- 可能没有正确的WHERE条件（total_score < 60）
- 权限控制不够严格

**解决方案：**
- 在生成SQL之前，先检测是否是"我的挂科记录"查询
- 如果是，直接使用固定的SQL查询
- 不依赖AI生成SQL

### 2. 为什么要表扬没有挂科的学生？

**原因：**
- 查询结果为空不应该显示"没有找到"
- "没有找到"会让学生感到困惑或焦虑
- 表扬可以增强学生的自信心
- 提供正向反馈，鼓励学生继续努力

### 3. 为什么要禁止教师和管理员查询？

**原因：**
- 挂科记录是学生的隐私数据
- 教师和管理员不应该通过聊天接口查询
- 应该使用专门的管理后台
- 保护学生隐私，符合数据安全要求
