## 📖 概述

本地Embedding服务使用 `Xenova/Transformers.js` 在本地运行embedding模型，**完全避免调用外部API**！

### ✨ 优势

- ✅ **完全免费** - 无需支付API费用
- ✅ **无限并发** - 不怕多用户同时访问
- ✅ **无网络延迟** - 本地计算，响应更快
- ✅ **无限流风险** - 完全自主控制
- ✅ **数据隐私** - 所有计算在本地进行

### ⚠️ 注意事项

- ⚠️ **首次加载慢** - 首次运行需要下载模型（约400MB）
- ⚠️ **内存占用** - 需要一定内存（约500MB-1GB）
- ⚠️ **CPU使用** - 计算embedding时需要CPU资源

## ⚙️ 配置方法

### 方式一：通过环境变量配置（推荐）

编辑 `backend/.env` 文件：

```env
# Embedding服务配置
# 模式说明：
# - api: 使用智谱AI Embedding API（需要付费，可能限流）
# - local: 使用本地embedding模型（完全免费，无限并发，推荐！）
EMBEDDING_MODE=local
```

### 方式二：代码中配置

在创建RAGService时指定：

```javascript
const ragService = new RAGService(apiKey, {
    embeddingMode: 'local'  // 使用本地模式
});
```

## ❓ 常见问题

### Q1: 首次加载模型很慢怎么办？

**A:** 这是正常的！首次运行需要下载模型文件（约400MB），之后会缓存到本地，下次启动就快了。

**模型缓存位置：**
- Windows: `C:\Users\你的用户名\.cache\huggingface\hub\`
- Linux/Mac: `~/.cache/huggingface/hub/`

### Q2: 内存占用太高怎么办？

**A:** 本地embedding服务需要约500MB-1GB内存。如果内存不足，可以考虑：

1. 使用API模式（但可能限流）
2. 增加服务器内存
3. 使用更小的模型（需要修改代码）

### Q3: 本地模式效果不如API怎么办？

**A:** 如果发现检索效果下降，可以尝试：

1. 调整相似度阈值（降低 `minScore`）
2. 增加 `topK` 值（返回更多结果）
3. 使用Rerank重排序（已默认启用）

### Q4: 如何切换回API模式？

**A:** 修改 `.env` 文件：

```env
EMBEDDING_MODE=api
```

然后重启服务即可。

### Q5: 两种模式可以混用吗？

**A:** 暂不支持。建议根据实际情况选择一种模式：
- 生产环境：**推荐本地模式**（稳定、免费）
- 开发测试：可以使用API模式（快速验证）

## 🆚 API vs 本地模式对比

### 详细对比表

| 特性 | API模式 | 本地模式 |
|------|---------|----------|
| **模型** | 智谱AI embedding-3 | Xenova/all-MiniLM-L6-v2 |
| **向量维度** | 1024 | 384 |
| **响应速度** | ~500ms（网络延迟） | ~100ms（本地计算） |
| **并发限制** | 有限制（约5-10 req/s） | 无限制 |
| **限流风险** | **高**（多用户时） | **无** |
| **成本** | 付费（20元/5000万tokens） | **完全免费** |
| **网络要求** | 必须联网 | 离线也能用 |
| **数据隐私** | 数据发送到第三方 | 完全本地 |
| **首次启动** | 即时可用 | 需下载模型（约400MB） |
| **维护成本** | 依赖第三方服务 | 自主维护 |
| **推荐场景** | 小规模应用 | **生产环境** |

### 本小姐的建议 💡

**🎯 生产环境 → 使用本地模式！**

理由：
1. 多用户场景不会限流
2. 长期来看成本更低
3. 响应速度更快
4. 数据更安全

**⚠️ 什么时候用API模式？**

1. 快速原型开发（不想等模型下载）
2. 内存受限的环境
3. 对向量精度要求极高的场景
