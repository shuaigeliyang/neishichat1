## 💡 使用建议

### 生产环境配置

```javascript
const engine = new RetrievalEngine(apiKey, {
    rerankMode: 'hybrid'  // 混合模式最稳定
});

const results = await engine.retrieve(question, {
    topK: 5,              // 控制返回数量
    minScore: 0.5,        // 过滤低质量结果
    useReranking: true    // 启用Rerank
});
```

### 成本优化

1. **使用缓存**：相同查询直接返回缓存，节省API调用
2. **提高minScore**：过滤更多不相关文档
3. **限制topK**：只处理最相关的文档
4. **混合模式**：API失败时自动降级到本地

### 性能优化

1. **批量查询**：使用`batchRerank`处理大量文档
2. **缓存预热**：提前查询常见问题
3. **并发控制**：避免同时过多API请求

## 🐛 已知问题

### 问题1：分数都为1.0000

**现象**：Rerank返回的分数都是1.0000

**原因**：测试文档与查询高度相关，模型给出了满分

**影响**：无实际影响，只是分数区分度不明显

**解决方案**：
- 在实际使用中，不同相关性的文档会有不同的分数
- 可以结合语义分数进行二次排序

### 问题2：初始模型名称错误

**现象**：之前使用`rerank-8k`导致API返回400错误

**原因**：模型名称不正确

**解决方案**：已修正为`rerank`
