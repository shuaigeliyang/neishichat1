# 下载功能修复报告 📥

> **修复工程师：** 内师智能体系统 (￣▽￣)ﾉ
> **修复日期：** 2026-03-20
> **问题类型：** CORS跨域错误 + 下载URL配置错误

---

## 🐛 问题描述

### 原始错误
```
Access to XMLHttpRequest at 'http://example.com/download' from origin 'http://localhost:5173'
has been blocked by CORS policy: Response to preflight request doesn't pass access control check:
Redirect is not allowed for a preflight request.
```

### 症状
- ❌ 点击下载按钮后出现CORS错误
- ❌ 请求的目标URL是 `http://example.com/download`（错误的URL）
- ❌ 无法下载任何Excel文件（个人信息、成绩单、班级信息、学院信息）

---

## 🔍 问题根源分析

### 1️⃣ **AI生成错误的下载URL**

**文件：** `backend/src/services/zhipuAI.js` 第172行

**问题提示词：**
```javascript
### 下载场景（返回特定JSON格式）
当用户说"下载"、"导出"时：
- JSON格式：{"type": "download", "downloadType": "类型", "downloadUrl": "URL", "message": "提示语"}
```

**问题分析：**
- 提示词要求AI生成 `downloadUrl: "URL"`
- 但**没有明确说明URL应该是什么格式**
- AI（智谱AI）编造了错误的URL：`http://example.com/download`

### 2️⃣ **前端缺少URL验证**

**文件：** `frontend/src/pages/Chat.jsx`

**问题：**
- 前端直接使用AI返回的`downloadUrl`
- 没有验证URL是否有效
- 导致发送请求到错误的地址

---

## ✅ 修复方案

### 修复1：移除AI生成下载URL的能力

**文件：** `backend/src/services/zhipuAI.js`

**修改内容：**
```javascript
// ❌ 修改前
### 下载场景（返回特定JSON格式）
当用户说"下载"、"导出"时：
- JSON格式：{"type": "download", "downloadType": "类型", "downloadUrl": "URL", "message": "提示语"}

// ✅ 修改后
### 下载场景
当用户说"下载"、"导出"时，**不要返回JSON，直接用自然语言回复**：
- 例如："好的！我已为您生成下载链接，请点击下方按钮下载Excel文件～"
- **注意：下载URL由后端自动生成，你不需要在JSON中提供URL**
```

**改进点：**
- ✅ AI不再生成下载URL
- ✅ 明确说明下载URL由后端处理
- ✅ 避免AI编造错误的URL

### 修复2：前端添加URL验证

**文件：** `frontend/src/pages/Chat.jsx`

**修改内容：**
```javascript
// ✅ 添加URL验证
else if (parsed.type === 'download') {
  // 验证downloadUrl是否有效（必须是/api/开头的相对路径）
  const downloadUrl = parsed.downloadUrl;
  if (!downloadUrl || !downloadUrl.startsWith('/api/')) {
    console.error('❌ 无效的下载URL:', downloadUrl);
    message.error('下载链接无效，请联系管理员');
    return;
  }

  return setMessages(prev => [...prev, {
    role: 'assistant',
    content: parsed.message,
    isDownload: true,
    downloadUrl: downloadUrl,
    downloadType: parsed.downloadType
  }]);
}
```

**改进点：**
- ✅ 验证URL必须以 `/api/` 开头
- ✅ 拒绝无效的URL（如 `http://example.com/download`）
- ✅ 显示友好的错误提示

## 🎯 下载功能完整流程

### 现在的工作流程

```
用户输入："下载个人信息"
    ↓
后端 chat.js 检测到"下载"关键词（第26-77行）
    ↓
后端自动生成正确的downloadUrl
- 个人信息：/api/export/student/profile
- 成绩单：/api/export/student/grades
- 班级信息：/api/export/class/{classId}
- 学院信息：/api/export/college/{collegeId}
    ↓
后端返回JSON给前端
{
  type: "download",
  downloadType: "个人信息",
  downloadUrl: "/api/export/student/profile",
  message: "✅ 已为您生成个人信息的下载链接..."
}
    ↓
前端验证URL（必须以/api/开头）
    ↓
前端显示下载按钮
    ↓
用户点击下载按钮
    ↓
前端发送请求到 /api/export/student/profile
    ↓
Vite代理转发到 http://localhost:3000/api/export/student/profile
    ↓
后端生成Excel文件并返回
    ↓
前端触发浏览器下载
```

## 🔧 技术细节

### 后端API端点

| 功能 | 方法 | 端点 | 权限 |
|------|------|------|------|
| 个人信息 | GET | `/api/export/student/profile` | 仅学生 |
| 成绩单 | GET | `/api/export/student/grades` | 仅学生 |
| 班级信息 | GET | `/api/export/class/:classId` | 学生（自己班级）/教师 |
| 学院信息 | GET | `/api/export/college/:collegeId` | 学生（自己学院）/教师 |

### 请求头

所有下载请求都需要携带认证token：

```javascript
headers: {
  'Authorization': `Bearer ${localStorage.getItem('token')}`
}
```

### 响应类型

```javascript
responseType: 'blob' // 重要：告诉axios返回二进制数据
```

## 🛡️ 安全性改进

### 1. URL验证
- 前端验证URL必须以 `/api/` 开头
- 防止请求到外部恶意URL

### 2. 权限控制
- 学生只能下载自己的数据
- 教师可以下载所教班级/学院的数据
- 后端进行二次权限验证

### 3. 认证token
- 所有下载请求都需要有效的JWT token
- 防止未授权访问

## 🎓 知识总结

### 教训1：不要让AI生成URL
AI（如智谱AI）可能会编造不存在的URL，应该由后端代码明确生成。

### 教训2：前端验证很重要
即使后端是正确的，前端也应该验证外部输入，防止安全问题。

### 教训3：完整的错误处理
友好的错误提示能帮助用户快速理解问题，也能帮助开发者快速定位bug。
