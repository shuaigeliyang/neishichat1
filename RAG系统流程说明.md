# RAG系统完整流程说明

## 1. 数据准备阶段

### 1.1 数据源
- **文件**: `student_handbook_full.json`
- **内容**: 128页学生手册（高质量，噪音少）
- **格式**: 
```json
{
  "total_pages": 4735,
  "pages": [
    {
      "page_num": 1,
      "text": "2025 STUDENT HAND BOOK\n..."
    }
  ]
}
```

### 1.2 智能分块（create_chunks_new.py）
**目的**: 将长文档分割成适合检索的小块

**策略**:
- 优先按段落分块（双换行符）
- 目标大小：1000字符
- 重叠大小：200字符
- 生成：252个文档块

**输出**: `document_chunks.json`
```json
{
  "metadata": {...},
  "chunks": [
    {
      "chunk_id": 1,
      "page_num": 1,
      "text": "...",
      "char_count": 1129,
      "full_context": "..."  // 用于embedding
    }
  ]
}
```

### 1.3 向量化（generate_embeddings_new.js）
**目的**: 为每个文档块生成向量表示

**过程**:
1. 使用智谱AI embedding-3模型
2. 为每个文档块的 `full_context` 生成2048维向量
3. 保存向量到 `embedding_cache.json`（252条缓存）
4. 保存完整索引到 `retrieval_index.json`

**输出**: `retrieval_index.json`
```json
{
  "metadata": {...},
  "chunks": [
    {
      "chunk_id": 1,
      "page_num": 1,
      "text": "...",
      "embedding": [2048维向量]
    }
  ],
  "indexed": true,
  "timestamp": "..."
}
```

## 2. 检索阶段

### 2.1 用户提问
**示例**: "我挂科了怎么办"

### 2.2 问题标准化（rag.js:normalizeQuestion）
**目的**: 将口语化表达转换为正式用语

**处理步骤**:
1. **主语补充**: "我挂科了怎么办" → "学生挂科了怎么办"
2. **同义词替换**: 
   - "挂科" → "不及格"
   - "怎么办" → "如何处理"
3. **最终**: "学生不成绩合格了如何处理"

**代码位置**: `backend/src/routes/rag.js` 第244-298行

### 2.3 向量化问题
**目的**: 将问题转换为向量

**过程**:
1. 使用智谱AI embedding-3模型
2. 生成问题的2048维向量
3. 检查缓存（如果已生成过则使用缓存）

**代码位置**: `backend/services/embeddingService.js`

### 2.4 相似度计算
**目的**: 找到与问题最相关的文档块

**算法**: 余弦相似度（Cosine Similarity）

**公式**:
```
similarity = (A · B) / (||A|| × ||B||)
```

其中A是问题向量，B是文档块向量

**代码位置**: `backend/services/embeddingService.js` 第172-188行

**输出**: 15个最相关的文档块（topK=15）

### 2.5 重排序（可选）
**目的**: 提高检索结果的准确性

**方法**:
- 优先使用智谱AI Rerank API
- API失败时降级到本地关键词匹配

**代码位置**: 
- `backend/services/retrievalEngine.js` 第138-259行
- `backend/services/enhancedLocalRerank.js`

**输出**: 重排序后的文档块

### 2.6 来源去重
**目的**: 同一页码只保留最相关的一个

**方法**: 使用Map按页码分组，保留评分最高的

**代码位置**: `backend/src/routes/rag.js` 第365-393行

**修复**: 使用 `source.page` 作为key（而不是 `${chapter}_${page}`）

## 3. 答案生成阶段

### 3.1 构建上下文
**目的**: 为AI提供参考文档

**格式**:
```
文档片段1（相关度：100%）：
[第85页] (一)警告；(二)严重警告...

文档片段2（相关度：90%）：
[第61页] 对学生作出留校察看...
```

**代码位置**: `backend/services/qaGenerator.js` 第143-152行

### 3.2 调用智谱AI生成答案
**模型**: glm-4-flash

**提示词**:
```
你是内江师范学院的学生助手，专门基于《学生手册》回答学生和辅导员的问题。

【核心原则】：
1. **准确性第一**：所有回答必须基于提供的文档片段
2. **清晰易懂**：用简单明了的语言解释政策和流程
...
```

**代码位置**: `backend/services/qaGenerator.js`

### 3.3 提取来源信息
**目的**: 为用户提供参考来源

**格式**:
```javascript
{
  chapter: "第85页",
  page: 85,
  score: 1.0,
  snippet: "..."
}
```

**代码位置**: `backend/services/qaGenerator.js` 第157-164行

### 3.4 格式化回答
**目的**: 添加来源标注

**格式**:
```
**回答内容**...

**参考资料**：
1. 第85页
2. 第61页
...
```

**代码位置**: `backend/services/qaGenerator.js` 第169-181行

**修复**: 使用 `${source.chapter}` 而不是 `${source.chapter}（第${source.page}页）`
