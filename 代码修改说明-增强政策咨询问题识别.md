## 📋 问题描述

### 错误现象

用户咨询政策相关问题，但都被识别为"普通聊天"：

| 用户问题 | 识别意图 | 实际意图 | 结果 |
|---------|---------|---------|------|
| 背处分还能评奖学金吗 | 普通聊天 ❌ | 文档问答 | 智谱AI回答，没有查询学生手册 |
| 挂科能毕业吗 | 普通聊天 ❌ | 文档问答 | 智谱AI回答，没有查询学生手册 |
| 补考难不难 | 普通聊天 ❌ | 文档问答 | 智谱AI回答，没有查询学生手册 |
| 重修要交钱吗 | 普通聊天 ❌ | 文档问答 | 智谱AI回答，没有查询学生手册 |
| 旷课会被抓吗 | 普通聊天 ❌ | 文档问答 | 智谱AI回答，没有查询学生手册 |

**用户反馈：** "这些全部是普通文本处理并没有返回相关的政策"

## 🔍 根本原因分析

### 问题流程

```
用户："背处分还能评奖学金吗"
  ↓
前端意图识别：
  1. 检查policyTopics：包含"处分"、"奖学金" ✅
  2. 检查inquiryKeywords：
     - "怎么办" ❌
     - "怎么" ❌
     - "如何" ❌
     - "能" ❌（没有单独的"能"）
     - "还能" ❌（没有单独的"还能"）
  ↓
  hasInquiryKeyword = false ❌
  ↓
  条件不满足：hasInquiryKeyword && hasPolicyTopic
  ↓
  降级到普通聊天 ❌
```

### 关键问题

**inquiryKeywords（咨询性关键词）不够全面：**

```javascript
// 当前的inquiryKeywords
const inquiryKeywords = [
  '怎么办', '怎么处理', '如何处理',
  '怎么申请', '如何申请',
  '会怎么样', '有什么影响',
  '后果', '结果', '是什么', '有哪些',
  '怎么拿', '如何获得',
  '怎么获得', '怎么评', '如何评选',
  '怎么', '如何',
  '能不能', '可不可以', '是否', '可'
  // ❌ 缺少："能...吗"、"难不难"、"要...吗"、"会被...吗"、"还能...吗"
];
```

**缺失的咨询性表达方式：**
- "能...吗"：能毕业吗、能评奖学金吗
- "难不难"：补考难不难、重修难不难
- "要...吗"：要交钱吗、要考吗
- "会被...吗"：会被抓吗、会被处分吗
- "还能...吗"：还能评奖学金吗、还能毕业吗

### 修改2：添加疑问句检测

**修改前：**

```javascript
// 检查是否同时包含咨询性关键词和政策相关词汇
const hasInquiryKeyword = inquiryKeywords.some(keyword =>
  question.toLowerCase().includes(keyword.toLowerCase())
);

const hasPolicyTopic = policyTopics.some(topic =>
  question.toLowerCase().includes(topic.toLowerCase())
);

// 🔒 特殊处理：排除陈述句模式

  console.log('🎯 意图识别：文档问答（政策规定）');

```

**修改后：**

```javascript
// 检查是否同时包含咨询性关键词和政策相关词汇
const hasInquiryKeyword = inquiryKeywords.some(keyword =>
  question.toLowerCase().includes(keyword.toLowerCase())
);

const hasPolicyTopic = policyTopics.some(topic =>
  question.toLowerCase().includes(topic.toLowerCase())
);

// 🔒 特殊处理：检测疑问句模式
// 如果问题以"吗"结尾，且包含政策词汇，识别为文档问答
const questionPattern = /吗[？?！!]*$/;
const isQuestion = questionPattern.test(question.trim());

// 🔒 特殊处理：排除陈述句模式
const statementPattern = /^我.+了$/;
const isStatement = statementPattern.test(question.trim());

// 如果同时满足以下条件之一，识别为文档问答：
// 1. 包含咨询性关键词和政策词汇，且不是陈述句
// 2. 以"吗"结尾的疑问句，且包含政策词汇
if ((hasInquiryKeyword && hasPolicyTopic && !isStatement) || (isQuestion && hasPolicyTopic)) {
  console.log('🎯 意图识别：文档问答（政策规定）');
  return 'document';
}
```

## 💡 设计思路

### 为什么要添加疑问句检测？

**原因：**
- 很多政策咨询问题以"吗"结尾
- "能...吗"、"难不难"、"会...吗"
- 这些问题不一定包含标准的咨询词（如"怎么办"、"怎么"）
- 但确实是疑问句，需要查询政策

**逻辑：**
```javascript
// 检测以"吗"结尾的疑问句
const questionPattern = /吗[？?！!]*$/;
const isQuestion = questionPattern.test(question.trim());

// 如果是疑问句，且包含政策词汇 → 文档问答
if (isQuestion && hasPolicyTopic) {
  return 'document';
}
```

### 为什么添加"吗"作为关键词？

**原因：**
- "吗"是中文疑问句的典型标志
- 配合policyTopics使用，不会误判
- 例如："你好吗" → 包含"吗"但不包含policyTopics → 普通聊天 ✅
- 例如："挂科能毕业吗" → 包含"吗"和"挂科" → 文档问答 ✅

## 🔍 边界情况处理

### 情况1：不包含政策词汇的疑问句

**问题：** "你好吗"、"吃饭了吗"

**检测：**
- isQuestion = true（以"吗"结尾）
- hasPolicyTopic = false（不包含政策词汇）

**结果：** 普通聊天 ✅

**说明：** 不会误判，因为没有政策词汇

---

### 情况2：包含政策词汇的陈述句

**问题：** "我挂科了"、"我受处分了"

**检测：**
- hasPolicyTopic = true（包含"挂科"）
- isStatement = true（符合"我.+了$"模式）

**结果：** 普通聊天 ✅

**说明：** 陈述句模式优先排除

---

### 情况3：复杂的疑问句

**问题：** "我想知道挂科了还能不能毕业"

**检测：**
- hasPolicyTopic = true（包含"挂科"）
- hasInquiryKeyword = true（包含"能不能"）

**结果：** 文档问答 ✅

**说明：** 包含咨询性关键词和政策词汇

## 🎯 总结

### 修改内容

1. **扩充咨询性关键词**
   - 添加："能吗"、"难吗"、"要吗"、"会吗"、"被"、"还"、"有没有"
   - 添加："吗"作为疑问句检测关键词

2. **添加疑问句检测**
   - 检测以"吗"结尾的疑问句
   - 配合policyTopics使用

3. **优化识别逻辑**
   - 条件1：hasInquiryKeyword && hasPolicyTopic && !isStatement
   - 条件2：isQuestion && hasPolicyTopic
   - 满足任一条件即识别为文档问答

### 解决的问题

- ✅ "背处分还能评奖学金吗" → 文档问答
- ✅ "挂科能毕业吗" → 文档问答
