## 🎯 修复的问题

**问题描述：** 用户说"我要下载转专业申请表"时，返回403 Forbidden错误。

**原因分析：**
1. ❌ 后端只接受 `templateId` 参数，但前端发送的是 `templateName`
2. ❌ 后端有权限限制：只允许学生生成表单，管理员被拒绝

**修复内容：**
1. ✅ 后端支持 `templateName` 参数（模糊匹配）
2. ✅ 移除权限限制，允许管理员和教师生成表单
3. ✅ 为管理员/教师使用示例学生信息

---

## ✅ 后端修改详情

### 文件：`backend/src/routes/forms.js`

**修改前：**
```javascript
router.post('/generate', authenticate, async (req, res) => {
  const { templateId } = req.body;  // 只接受templateId

  if (userType !== 'student') {
    return error(res, '只有学生才能生成表单', 403);  // 权限限制
  }
  // ...
});
```

**修改后：**
```javascript
router.post('/generate', authenticate, async (req, res) => {
  const { templateId, templateName } = req.body;  // 支持两种参数

  // 支持templateName模糊匹配
  if (templateName) {
    template = await query(`
      SELECT * FROM form_templates
      WHERE template_name LIKE ? AND status = 1
    `, [`%${templateName}%`]);
  }

  // 允许所有用户生成表单
  if (userType === 'admin' || userType === 'teacher') {
    // 使用示例学生信息
    studentInfo = [{ /* 示例信息 */ }];
  }
  // ...
});
```

**新增功能：**
1. 支持两种参数传递方式：
   - `templateId`：精确匹配（例如：1）
   - `templateName`：模糊匹配（例如："转专业申请表"）

2. 权限放宽：
   - 学生：自动填充真实个人信息
   - 管理员/教师：使用示例信息

3. 支持的表单类型：
   - ✅ 学科竞赛参赛申请表
   - ✅ 转专业申请表
   - ✅ 奖学金申请表
   - ✅ 休学申请表
   - ⚠️ 其他表单类型：开发中

```javascript
  templateName: '转专业申请表'

```

**方式2：使用templateId**
```javascript
// 前端代码
axios.post('/api/forms/generate', {
  templateId: 2  // 假设转专业申请表的ID是2
});
```

### 后端查询逻辑

```javascript
// templateName模糊匹配
if (templateName) {
  template = await query(`
    SELECT * FROM form_templates
    WHERE template_name LIKE ? AND status = 1
  `, [`%${templateName}%`]);
}

// 示例：
// templateName: "转专业"  → 匹配 "转专业申请表"
// templateName: "竞赛"    → 匹配 "学科竞赛参赛申请表"
// templateName: "奖学金"  → 匹配 "奖学金申请表"
```
