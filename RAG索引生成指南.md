# RAG 索引文件生成指南

## 问题说明

由于 `retrieval_index.json` 文件过大（58MB），未包含在仓库中。首次运行时需要自动生成该文件。

## 自动生成

**好消息！** RAG 系统已经实现了自动检测和生成功能：

1. 首次启动后端服务时，系统会自动检测索引文件是否存在
2. 如果不存在，系统会自动从 `document_chunks.json` 重新生成索引
3. 生成过程可能需要几分钟，请耐心等待

## 手动生成（可选）

如果需要手动重新生成索引，可以运行：

```bash
cd backend
node rebuild-index.js
```

## 文件说明

- **document_chunks.json** (4.1MB) - ✅ 已包含
  - 文档分块数据，RAG 系统的基础数据

- **retrieval_index.json** (58MB) - ❌ 未包含（自动生成）
  - 向量索引文件，首次运行时自动生成
  - 包含所有文档块的向量化表示

- **embedding_cache.json** - 自动生成
  - 向量缓存，用于加速查询

## 注意事项

⚠️ **首次运行需要较长时间**
- 索引生成需要调用智谱 AI API
- 确保已正确配置 `.env` 文件中的 `ZHIPU_API_KEY`
- 生成过程大约需要 5-10 分钟（取决于网络和 API 速度）

✅ **生成后会自动保存**
- 索引生成后会保存到 `retrieval_index.json`
- 下次启动时会直接加载，无需重新生成

## 故障排除

### 问题：索引生成失败

**解决方案：**
1. 检查 API 密钥是否正确配置
2. 检查网络连接是否正常
3. 查看控制台错误日志

### 问题：生成时间过长

**解决方案：**
1. 正常现象，请耐心等待
2. 可以在控制台查看生成进度
3. 如长时间无响应，检查 API 配置

## 技术细节

索引生成过程：

1. **加载文档块** - 从 `document_chunks.json` 加载所有文档
2. **向量化** - 使用智谱 AI 的 Embedding API 将文本转换为向量
3. **构建索引** - 建立向量索引，支持快速检索
4. **保存索引** - 将索引保存到 `retrieval_index.json`
