---

## 🎯 问题描述

**原来的问题：**
```
用户提问
    ↓
1️⃣ 尝试智能查询（等待响应...）
    ↓ 失败（403）
2️⃣ 尝试RAG文档问答（等待响应...）
    ↓ 失败
3️⃣ 使用普通聊天
```

**问题分析：**
- ❌ 顺序尝试浪费时间
- ❌ 每次失败都要等待超时
- ❌ 用户体验差

---

## ✅ 优化方案

### 添加智能意图识别

在发送请求前，先识别用户意图，直接调用对应的接口！

```
用户提问
    ↓
🎯 意图识别（瞬间完成）
    ↓
    ├─ database → 数据库查询
    ├─ document  → RAG文档问答
    └─ chat      → 普通聊天
    ↓
立即调用对应接口
```

### 意图识别规则

**1. 数据库查询（个人数据）**
```
关键词：我的、本人、自己、我的成绩、我的课程等

示例：
- "我的成绩是多少？"
- "我的课表是什么？"
- "我的个人信息"
```

**2. 文档问答（政策规定）**
```
关键词：规定、办法、流程、如何、重修、奖学金等

示例：
- "重修需要什么条件？"
- "如何申请奖学金？"
- "转专业的流程是什么？"
```

**3. 普通聊天（其他）**
```
不符合上述两类的问题

示例：
- "你好"
- "今天天气怎么样"
- "讲个笑话"
```
