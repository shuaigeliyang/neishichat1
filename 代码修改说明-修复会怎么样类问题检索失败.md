## 🔍 根本原因分析

### 问题流程

```
用户："挂科了会怎么样"
  ↓
标准化：/"挂科" → "不及格"/ → "不及格了会怎么样"
  ↓
向量化：生成"不及格了会怎么样"的向量
  ↓
文档向量：["课程考核不合格可允许补考或重修"]
  ↓
相似度计算：cosine_similarity = 0.45（假设）
  ↓
❌ 关键问题：embeddingService.js第138行硬编码过滤
  if (score > 0.5) {  // 硬编码的0.5阈值！
    保留
  } else {
    过滤掉 ❌
  }
  ↓
检索结果：0个文档
```

### 关键问题

**问题1：硬编码的相似度阈值**

在`embeddingService.js`的第138行：
```javascript
.filter(item => item.score > 0.5);  // 过滤低相似度结果
```

**问题：**
- 阈值硬编码为0.5
- 即使在rag.js中设置`minScore: 0.3`也没用
- 因为`findMostSimilar`内部已经用0.5过滤了
- "不及格了会怎么样"与"课程考核不合格可允许补考或重修"的相似度可能是0.45
- 0.45 < 0.5，所以被过滤掉了

**问题2：口语化表达未映射**

在`rag.js`的`SYNONYM_MAP`中：
```javascript
// ❌ 缺少这些映射：
'会怎么样' → '后果'
'怎么样' → '后果'
'有什么影响' → '影响'
```

**后果：**
- 用户说"会怎么样"
- 文档中说"后果"或"影响"
- 语义距离远
- 相似度低

## 🔧 修改内容

### 修改1：embeddingService.js - 参数化相似度阈值

**文件：** `backend/services/embeddingService.js`
**位置：** 第123-141行（findMostSimilar函数）

**修改前：**

```javascript
/**
 * 找到最相似的文档块
 */
async findMostSimilar(questionEmbedding, chunks, topK = 5) {
    const similarities = chunks.map(chunk => {
        const embedding = chunk.embedding;
        if (!embedding) {
            return { chunk, score: 0 };
        }

        const score = this.cosineSimilarity(questionEmbedding, embedding);
        return { chunk, score };
    });

    // 排序并取Top-K
    const topKResults = similarities
        .sort((a, b) => b.score - a.score)
        .slice(0, topK)
        .filter(item => item.score > 0.5);  // ❌ 硬编码的0.5阈值

    return topKResults;
}
```

**修改后：**

```javascript
/**
 * 找到最相似的文档块
 * ✨ 修改：添加minScore参数，允许外部传入阈值，默认从0.5降低到0.3
 */
async findMostSimilar(questionEmbedding, chunks, topK = 5, minScore = 0.3) {
    const similarities = chunks.map(chunk => {
        const embedding = chunk.embedding;
        if (!embedding) {
            return { chunk, score: 0 };
        }

        const score = this.cosineSimilarity(questionEmbedding, embedding);
        return { chunk, score };
    });

    // 排序并取Top-K
    const topKResults = similarities
        .sort((a, b) => b.score - a.score)
        .slice(0, topK)
        .filter(item => item.score >= minScore);  // ✨ 使用传入的minScore参数

    return topKResults;
}
```

**修改说明：**
- ✅ 添加`minScore`参数，默认值为0.3（比原来的0.5更宽松）
- ✅ 将硬编码的0.5改为使用传入的`minScore`参数
- ✅ 将`> 0.5`改为`>= minScore`，包含边界值

## 💡 设计思路

### 为什么要降低minScore？

**原因：**
- 口语化问题与正式文档的语义距离较远
- 向量模型的局限性：无法完全理解所有口语化表达
- 降低阈值可以召回更多相关文档
- 通过Rerank重新排序，保证质量

**权衡：**
- minScore太低（如0.1）：召回很多无关文档，噪音大
- minScore太高（如0.5）：召回太少，漏掉相关文档
- minScore=0.3：平衡点，既能召回又能保证质量

### 为什么要添加"会怎么样"映射？

**原因：**
- 用户常说"会怎么样"，文档常说"后果"或"影响"
- 向量模型可能无法自动关联这两个词
- 显式映射可以缩小语义距离

**效果：**
- "会怎么样" → "后果"
- "不及格了会怎么样" → "不及格了后果"
- 更接近文档中的"课程考核不合格可允许补考或重修"

## 🔍 边界情况处理

### 情况1：相似度在0.3-0.5之间的文档

**问题：** 以前会被过滤掉，现在能召回

**结果：**
- ✅ 召回更多相关文档
- ✅ 通过Rerank重新排序，高质量文档排在前面
- ⚠️ 可能召回一些低质量文档，但可以通过Rerank过滤

### 情况2：相似度低于0.3的文档

**问题：** 即使降低阈值也找不到了

**结果：**
- 这些文档确实与问题无关
- 不召回是正确的
- 系统会返回"没有找到"，自动降级到智谱AI
